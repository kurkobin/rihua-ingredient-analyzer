### 1. Demo 简介

**是什么**:"成分扫一扫" 是一款网页应用(Web App),用户用手机或电脑上传商品背面配料表图片,AI 自动识别成分并分析出产品的优缺点、风险等级与综合评分。

**面向谁**:注重健康的 25-40 岁城市消费者、过敏体质/敏感肌人群、成分党、为儿童选购洗护用品的家长。

**核心功能**(3 个):
1. **扫一扫配料表**:上传商品配料表图片,OCR 自动识别文字(支持中英混合)
2. **成分分析与优缺点提炼**:自动分类成分(表面活性剂/防腐剂/活性物等),标注风险等级(安全/注意/慎用/规避),生成"优点 N 条 / 缺点 N 条"
3. **综合评分与可读化呈现**:0-100 评分 + 成分雷达图维度 + 一句话总结,把专业成分翻译成人话

### 2. Demo 创作思路

**灵感来源**:
起初我在购买洗发水时,发现成分表里有"月桂醇硫酸酯钠(SLS)"。我知道这个成分清洁力极强,但刺激性也很大,这让我对自己的健康产生了担忧。后来买牙膏时,又看到了同样的 SLS 成分——两个都是清洁力极强的成分,那对牙齿是否有隐性或慢性伤害?这让我很难抉择。最终我在多方对比参考下,购买了不含这些成分的产品。

这个过程让我意识到:普通消费者根本看不懂配料表,而商家宣传又往往回避成分问题。如果有一个工具能"扫一扫就让客户知道自己买的产品到底是什么样的",就能解决这个信息不对称。

**想解决的问题**:
- 配料表专业难懂,普通消费者无法判断产品好坏
- 商家宣传与实际成分不符(如标榜"温和"却含强刺激成分)
- 同类产品难对比,消费者决策成本高

**为什么做这个方向**:
我调研发现,市面上的同类产品(Yuka、美丽修行、真我 APP)要么只做单一品类(只化妆品或只食品),要么依赖条形码数据库(小众商品查不到),真正做"OCR 扫配料表 + 跨品类 + 优缺点分析"的产品几乎是空白。这是一个有真实需求且存在市场空白的方向。

### 4. TRAE 实践过程

**开发工具**:TRAE IDE

**完整开发流程**:

1. **需求调研阶段**:用 TRAE 的 WebSearch 工具调研市面同类产品(Yuka、美丽修行、真我 APP 等),输出竞品对比分析与市场空白点

5. **调试优化**:在 TRAE 中排查 OCR 超时、环境变量读取等问题,完成迁移与部署

## 开发心得(可选补充)

1. **OCR 是核心入口,但不是全部**:初期我只想着识别文字,后来发现真正的价值在于"把专业成分翻译成人话",LLM 在这里发挥了关键作用
2. **成分库的冷启动难题**:MVP 阶段聚焦日化洗护核心 30 种成分,用 LLM 兜底未入库成分,避免了"查不到"的尴尬
3. **大图片的超时坑**:用户上传的配料表照片通常 3-4MB,导致 OCR 接口超时,需调整超时阈值
