## 二、市场可行性

### 2.1 需求真实性 ✅
- 用户痛点真实:成分表专业难懂、商家宣传与实际不符、健康焦虑上升
- 搜索热度:"成分党"、"配料表查询"持续增长
- 用户场景明确:超市现场决策、网购前对比、回购决策

### 2.2 竞争格局 ✅
- 头部产品(Yuka)海外为主,中文商品覆盖差
- 国内产品多专注单一品类(化妆品 OR 食品)
- OCR 扫配料表 + 跨品类 + 优缺点分析 = 差异化空间

### 2.3 风险 ⚠️
- 巨头入场风险:美丽修行、真我可能扩展品类
- 用户教育成本:让用户养成"扫码看成分"习惯需时间
- 同质化:技术壁垒不高,易被复制

---

## 三、技术可行性

### 3.1 OCR 识别 ✅
- **成熟度**:高。PaddleOCR 中文识别率达 90%+;腾讯云/阿里云 OCR 95%+
- **挑战**:配料表排版复杂(小字、换行、中英混排、特殊符号),需后处理
- **成本**:开源免费;云服务约 0.01-0.05 元/次

### 3.2 成分匹配 ✅
- INCI 名称标准化、食品添加剂 GB 标准化
- 难点:同义词归一(如"月桂醇硫酸酯钠" = SLS = Sodium Lauryl Sulfate)
- 解决:构建同义词词典 + LLM 辅助归一

### 3.3 大模型分析 ✅
- LLM(DeepSeek/通义/GPT)可生成优缺点、解释成分作用
- **关键**:Prompt 工程 + 知识库 RAG,避免幻觉
- 成本:DeepSeek 约 0.001-0.01 元/次,可控

### 3.4 移动端 APP ✅
- Flutter / RN 跨平台,一套代码双端
- 相机、相册、扫码 SDK 均成熟

### 3.5 技术风险 ⚠️
- OCR 在低光、模糊、曲面包装上识别率下降 → 引导用户重拍 + 可编辑校正
- LLM 幻觉 → 必须叠加规则引擎校验,不能纯靠 LLM
- 成分数据库覆盖度 → 长期工程,需持续补数据

## 四、数据可行性

### 4.1 可获取数据源 ✅
| 数据 | 来源 | 获取方式 |
|---|---|---|
| 化妆品成分 | 药监局备案、CosIng、EWG | 公开爬取/API |
| 食品添加剂 | GB 2760、FDA GRAS | 公开标准 |
| 商品条码 | 中国商品信息中心、Open Food Facts | API/众包 |
| 成分风险 | CIR、EFSA、文献 | 整理入库 |

### 4.2 数据挑战 ⚠️
- **冷启动**:初期成分库覆盖不全,小众商品可能查不到
- **数据质量**:多源数据需清洗去重,同名异义/异名同义问题
- **更新维护**:法规更新、新成分上市,需持续维护
- **成本**:自建成分库初期需 1-3 个月人工整理

### 4.3 应对策略
- MVP 阶段聚焦洗发水、牙膏、面霜、零食等 Top 20 品类,成分库覆盖 500-1000 种即可起步
- 引入用户纠错机制,众包补数据
- LLM 兜底:未入库成分也能给出基础说明

## 五、商业可行性

### 5.1 变现路径

| 模式 | 说明 | 可行度 |
|---|---|:---:|
| **导购佣金** | 推荐替代品,带电商 CPS 链接 | ★★★★☆ |
| **会员订阅** | 高级分析、个性化报告、无限扫描 | ★★★☆☆ |
| **品牌合作** | 品牌方数据接入、精准营销 | ★★★☆☆(需规模) |
| **B 端 API** | 给零售商/平台提供成分分析能力 | ★★☆☆☆(后期) |
| **广告** | 同类商品广告 | ★★☆☆☆(影响体验) |

### 5.2 商业风险 ⚠️
- 早期无规模,导购佣金收入有限
- 负面评价可能招致品牌方压力
- 用户付费意愿需验证(国内工具类付费转化率低)

### 5.3 建议路径
1. 早期:免费 + 导购佣金,积累用户
2. 中期:推出会员,验证付费
3. 长期:B 端 API + 品牌服务

## 六、法律合规可行性

### 6.1 主要风险点 ⚠️⚠️

| 风险 | 说明 | 应对 |
|---|---|---|
| **品牌名誉权** | 负面评价可能被诉 | 数据有据可依、标注来源、免责声明 |
| **广告法** | 不得使用"最"、"虚假"等绝对化用语 | 文案审核机制 |
| **医疗暗示** | 不得暗示治疗、预防疾病 | 仅做成分分析,不提疗效 |
| **数据版权** | 爬取备案/条码数据的合法性 | 使用公开 API、遵循 robots |
| **个人信息** | 用户画像合规 | 隐私政策、最小化采集、PIPL 合规 |
| **食品安全法** | 不得造谣食品不安全 | 客观陈述、引用标准条款 |

### 6.2 关键合规动作
- 上线前过法务:隐私政策、用户协议、免责声明
- 评分与结论必须有权威依据(国标、文献),可溯源
- 建立品牌方申诉通道
- 避免主观定性("这是毒药"),用客观表述("含 XX,可能引起 XX")

## 九、可行性结论

**结论:项目可行,建议启动。**

核心依据:
1. 用户痛点真实,市场存在空白(跨品类 + OCR + 优缺点)
2. 核心技术(OCR、LLM、移动端)均已成熟,门槛大幅降低
3. 启动成本低,单人/小团队即可验证 MVP
4. 变现路径清晰,可逐步推进

**关键成功因素:**
- 数据准确性与权威性(决定信任度)
- 用户体验(扫描速度、结果可读性)
- 品类覆盖广度(决定使用频次)

**建议起步策略:** 先做 Web 版 MVP,用最小成本验证"扫描 → 分析 → 展示"闭环与用户价值,再决定是否投入 APP 开发。
