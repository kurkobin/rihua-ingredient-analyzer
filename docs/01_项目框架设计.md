# 商品成分分析 APP — 项目框架设计

> 版本:v0.1  日期:2026-07-08
> 核心目标:扫一扫商品背面配料表,自动分析并展示产品优缺点,帮助消费者看清商品"真实性"

---

## 一、市场竞品对比

### 1. 国内同类产品

| 产品 | 领域 | 核心方式 | 优势 | 不足 |
|---|---|---|---|---|
| **真我 APP** | 化妆品 | 名称搜索 + 成分数据库(2.7万成分) | 数据权威、备案同步、皮肤匹配 | 偏搜索,无 OCR 扫配料表 |
| **美丽修行** | 化妆品 | 名称搜索 + 安全评分(350万商品) | 评分体系成熟、社区活跃 | 仅化妆品,扫码为辅 |
| **透明标签** | 化妆品 | 成分标签检索 | 简洁清爽 | 数据规模较小 |
| **化妆品监管 APP** | 化妆品 | 官方条形码扫码查备案 | 官方权威、可查真伪 | 不分析成分优劣 |
| **食探配料查** | 食品 | 扫码 + 手动输入 | 标注过敏原与添加剂 | 仅食品,无优缺点分析 |
| **配料查 APP** | 食品 | 条形码 + 扫描 | 食品添加剂齐全 | 仅食品,无综合评分 |
| **扫描配料表 APP** | 食品 | 条形码 + DeepSeek R1 分析 | 引入大模型 | 仅食品,数据准确度待验证 |
| **查配料** | 食品 | OCR 识别配料表 | 真·拍照 OCR | 仅食品,功能单一 |

### 2. 国外同类产品

| 产品 | 领域 | 核心方式 | 优势 | 不足 |
|---|---|---|---|---|
| **Yuka**(法国) | 食品 + 化妆品 | 扫条形码 + 健康评分(0-100) | 全球标杆、用户超千万 | 不支持中文商品、无 OCR |
| **Think Dirty** | 化妆品 | 扫码 + Dirty Meter 评分 | 毒理维度独特 | 仅化妆品、海外数据 |
| **EWG Healthy Living** | 食品 + 化妆品 | 条形码 + EWG 评分(1-10) | NGO 背书、标准透明 | 数据偏北美 |
| **Edo** | 食品 | 扫码 + 健康打分(0-10) | 算法简洁 | 已被 Yuka 蚕食 |
| **Glo Skin Scanner** | 护肤 | 拍照分析 | AI 肤质推荐 | 仅护肤 |
| **INCIdecoder** | 化妆品 | INCI 名称解析 | 专业 | 无扫码、偏工具 |

### 3. 市场空白点(本项目机会)

1. **跨品类综合分析**:目前国内"食品 + 日化(洗发水/牙膏) + 化妆品"全覆盖的产品几乎空白,绝大多数只做单一品类。
2. **OCR 扫配料表**:多数产品依赖条形码(依赖第三方数据库),对未入库的小众/进口商品无效;真正做 OCR 识别配料表的国内产品极少且功能单一。
3. **优缺点双维度展示**:现有产品多给"安全评分",缺少"优点 / 缺点 / 替代建议"这种更易理解的呈现方式。
4. **真实性问题**:鲜有产品针对"宣传成分 vs 实际成分"、"备案成分 vs 标注成分"做对比核验。
5. **个性化提醒**:针对用户过敏史、肤质、孕期等个性化风险提示较弱。

## 二、产品定位

### 一句话定位
> **"扫一扫配料表,看清商品真实的样子"** — 跨品类的商品成分分析与真实性核验工具。

### 目标用户
- 25-40 岁注重健康的城市消费者
- 过敏体质 / 敏感肌 / 孕产人群
- 成分党、理性消费者
- 家长(为儿童选购食品/洗护)

### 核心价值主张
1. **看清**:OCR 扫描配料表,不依赖条形码数据库,任何商品都能分析
2. **看懂**:把专业成分翻译成"优点 / 缺点 / 风险"的人话
3. **看清真假**:与备案/国标对比,核验宣传与实际是否一致
4. **看明白**:给出同类更优替代品建议

## 四、核心功能模块

### M1 扫一扫(核心入口)
- 拍照 / 相册导入配料表图片
- 自动 OCR 识别文字(中英混合)
- 条形码扫描(辅助,作为快速匹配入口)
- 识别结果可编辑校正

### M2 成分分析(核心大脑)
- 成分分类:活性成分 / 表面活性剂 / 防腐剂 / 香精 / 防晒剂 / 营养素 / 添加剂...
- 风险评级:安全 / 注意 / 慎用 / 规避
- 优缺点提炼:自动生成"优点 N 条 / 缺点 N 条"
- 个性化提醒:基于用户画像(过敏、孕产、儿童)

### M3 真实性核验
- 与药监局备案成分对比
- 与国标(GB)允许使用清单对比
- 宣传词 vs 实际成分一致性检查(如"无硅油"是否真的无)
- 异常标注

### M4 商品详情
- 综合评分(0-100)
- 成分雷达图(清洁力 / 温和度 / 安全性 / 功效性 / 性价比)
- 同类排名
- 用户评价聚合

### M5 替代推荐
- 同品类中更优商品推荐
- "无 XX 成分"专项筛选
- 价格 / 渠道链接(可选)

### M6 用户中心
- 个人画像(肤质 / 过敏原 / 关注点)
- 扫描历史
- 收藏 / 对比清单
- 反馈纠错(贡献数据)

## 五、技术选型(建议)

| 层级 | 技术选型 | 说明 |
|---|---|---|
| 移动端 | **Flutter** 或 **React Native** | 一套代码双端,适合早期快速验证;后期可转原生 |
| Web 端 | React + Vite + Tailwind | 用于网页版展示与后台管理 |
| 后端 | Python (FastAPI) 或 Node.js (NestJS) | Python 利于接入 ML/LLM |
| OCR | PaddleOCR / 腾讯云 OCR / 阿里云 OCR | PaddleOCR 开源免费,云服务准确率更高 |
| 条形码 | ML Kit / ZXing | 移动端本地识别 |
| 数据库 | PostgreSQL + Redis | 关系数据 + 缓存 |
| 大模型 | DeepSeek / 通义千问 / GPT-4o | 成分语义分析、优缺点生成 |
| 对象存储 | 阿里云 OSS / 腾讯云 COS | 存储扫描图片 |
| 部署 | Docker + Nginx | 早期单机即可 |

## 六、数据来源

### 成分数据库
- 化妆品:国家药监局备案成分库、CosIng(欧盟)、EWG Skin Deep
- 食品:GB 2760 食品添加剂使用标准、FDA GRAS
- 日化:相关国标 + 行业公开数据

### 商品数据库
- 条形码:中国商品信息中心、开放食品 Facts API、Barcode Lookup
- 备案:药监局国产/进口化妆品备案
- 用户贡献:扫码纠错、众包补充

### 风险规则
- 权威机构清单:国家药监局、EFSA、CIR、FDA
- 文献:PubMed、知网(关键词:致敏、致痘、内分泌干扰)
